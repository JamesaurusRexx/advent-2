# Advent 2

Advent 2 is a single-rocket project repository for design, construction, testing, flight preparation, and post-flight analysis.

## Project map

- [Design](design/README.md) — requirements, architecture, interfaces, and major decisions
- [Airframe](airframe/README.md) — body, fins, nose cone, couplers, and structural details
- [Propulsion](propulsion/README.md) — certified motor selection, retention, integration, and records
- [Avionics](avionics/README.md) — electronics, wiring, firmware, batteries, and sensors
- [Recovery](recovery/README.md) — parachutes, harnesses, deployment, and packing procedures
- [Simulations](simulations/README.md) — flight models, assumptions, and predicted performance
- [Manufacturing](manufacturing/README.md) — bills of materials, build procedures, and inspections
- [Testing](testing/README.md) — test plans, acceptance criteria, and results
- [Flight data](flight-data/README.md) — configurations, weather, raw data, photos, and reports
- [Documentation](documentation/README.md) — checklists, meeting notes, references, and change history

## Suggested workflow

1. Open a GitHub Issue for each task, question, defect, or test.
2. Record important design decisions and their reasoning in `design/`.
3. Keep units, dates, part numbers, revisions, and data sources with every result.
4. Preserve original test and flight data; put analysis in separate files.
5. Update the relevant subsystem README when its design changes.

## Project status

## Safety

Follow applicable laws, launch-site rules, and recognized rocketry safety codes. Use appropriately certified commercial motors and qualified supervision. Repository documentation is not a substitute for a formal safety review or launch authorization.
